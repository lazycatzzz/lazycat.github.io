JavaScript深入理解——对象

对象我相信是JavaScript非常常见的基本类型，不管是我们日常用来存储键值对的对象，还是我们的函数，它们都是对象。最近看书了解到了对象一些从未了解过的名词，在这探讨一下。

什么是对象？

JavaScript的设计是一个简单的基于对象的范式。一个对象就是一系列属性的集合，一个属性包含一个名和一个值。一个属性的值也可以是函数（函数的引用，或称方法）。

语法

对象有两种定义方式：声明（文字）形式和构造形式。

// 声明形式
var obj = {
  a: 2
}

//构造形式
var myObj = new Object()
myObj.a = 2

两种形式创建出来的对象都是一样的。不一样的是创建的行为。通过声明形式创建对象，可以添加多个键值对，而构造形式则需要逐个添加。所以一般我们都通过声明形式创建对象。

类型

JavaScript有六大（或七大）主要类型，分别是：

  • string
  • number
  • boolean
  • undefined
  • null
  • object

其中，string、number、boolean、undefined、null 都是简单基本类型，它们不是对象。我们可以通过typeof来获取它们的类型。

注意：typeof null的返回值是”object“，但这并不代表它就是对象类型，它只是简单基本类型。之所以会返回object，是因为JavaScript语言的一个bug。原理是这样的，不同的对象在底层都表示为二进制，在JavaScript中二进制前三位都为0的话就会被判断为object类型，null的二进制表示是全0，前三位自然也是0，所以typeof执行返回的就是“object”。

内置对象

JavaScript内部还有一些对象子类型，通常被称为内置对象。

  • String
  • Number
  • Boolean
  • Object
  • Function
  • Array
  • Date
  • RegExp
  • Error

这些对象通过new来调用，也就相当于是构造函数。

var str = "It is string"
typeof str // "string"

var strObj = new String("It is stirng")
typeof strObj // "object"

“It is string”只是一个字面量，它没有属性。但是我们很多时候可以对其进行一些操作，比如获取长度，访问某个字符等。这是因为语言在必要时自动把字符串字面量转换成一个String对象，也就是说我们不用显式地创建一个对象。通过这样我们就可以访问String对象上的属性和方法。

所以在使用字面量时，我们不需要显示地创建相应对象。

null 和 undefined 没有对应的构造形式，它们只有文字形式。相反，Date 只有构造，没有 文字形式。

Error对象很少在代码中显式创建，一般都是在抛出异常时自动创建。但我们也可以用new来创建一个。当然一般都用不着这样。

对象的内容

对象的内容是由一些存储在特定命名位置的（任意类型的）值组成的，我们称之为属性。

需要注意的时，我们说”内容“时，似乎在暗示它们就是存储在对象内部的。但这只是它的表现形式。在引擎内部，这些值的存储方式是多种多样的，存储在对线内部的只是这些属性的名称，它们就像指针，指向这些值真正存储的位置。

我们可以通过 . 操作符或者 [] 操作符来访问对象中的属性。

var obj = {
	a: 2
}
obj.a // 2
obj[a] // 2

.a 语法通 常被称为“属性访问”，[“a”] 语法通常被称为“键访问”。

两种访问方式是有区别的，主要区别在于，. 操作符要求属性名符合标识符的命名规范并且是一个确定的值，而[] 操作符可以接受任意UTF-8/Unicode 字符串作为属性名。举个例子，如果引用名是”super-Fun“的属性，那么就必须用[”super-Fun“]语法访问。不仅如此，[..] 内也可以是一个变量，在有的场景下，不确定的变量名我们用 [] 来访问。

var obj = {
  a: 2,
  b: 3
}

var str = 'a'
obj[str] // 2

str = 'b'
obj[str] // 3

我总结为” . 能做的，[] 也能做，[] 能做的，. 不一定能做“

在对象中，属性名永远是字符串，所以，如果我们使用string以外的其他值作为属性名，那它首先会被转换为一个字符串。

var obj = {}

obj[true] = "foo"
obj[3] = "bar"
obj[obj] = "baz"

obj["true"] // "foo"
obj["3"] // "bar"
obj["[object Object]"] // "baz"

可计算属性名

ES6增加了可计算属性名，可以在文字形式中使用 [] 包裹一个表达式来当作属性名

var prefix = "foo"
var myObject = {
  [prefix + "bar"]: "hello",
  [prefix + "baz"]: "world"
}

myObject["foobar"] // "hello"
myObject["foobaz"] // "world"

可计算属性名最常用的场景可能是在ES6的符号（Symbol）。简单来说，Symbol就是一种新的基本数据类型，包含一个不透明且无法预测的值（从技术角度来说就是一个字符串）。一般来说我们不会用到符号的实际值（不同JavaScript引擎中值是不同的），所以通常接触到的是JavaScript符号的名称，它们代表了内部语言行为。这里不详细介绍。

数组的特殊用法

这个仅作了解，不推荐使用。

一般我们使用数组就是通过索引值来访问存储的值。但数组也是对象，所以虽然每个下标都是整数，我们仍然可以给数组添加属性。

var array = ["foo", 42, "bar"]
array.baz = "baz"
array.length // 3
array,baz // "baz"

可以看到我们添加了新的属性在数组上并且也访问到了，数组的长度也是没有改变。

这种行为就跟我们使用对象一样，那我们可以用数组来做对象的行为？这并不是一个好主意，数组和普通对象都根据其对应的行为和用途进行了优化，所以最好只用对象来存储键值对，只用数组来存储数组下标值对。

属性描述符

在ES5之前，JavaScript语言本身并没有提供可以直接检测属性特性的方法，从ES5开始，所有属性都具备了属性描述符。

var obj = {
	a: 2
}

Object.getOwnPropertyDescriptor(obj, "a")
// {
//   value: 2,
//   writable: true,
//   enumerable: true,
//   configurable: true
// }

这就是对象属性对应的属性描述符（也可以称为“数据描述符”），它里面有它的值value以及三个特性：writable(可写)、enumerable(可枚举)、configurable(可配置)

在创建普通属性时属性描述符会使用默认值，我们可以通过Object.defineProperty(..)来添加一个新属性或者修改一个已有属性，并对其特性进行设置。

var obj = {}

Object.defineProperty(obj, "a", {
	value: 2,
	writable: true,
	configurable: true,
	enumerable: true
})

obj.a // 2

可以看到我们通过这种方式创建了一个属性，我们还可以对其特性进行修改，不过先了解每个特性的用途。

writable

writable决定属性是否可以被修改

var obj = {}

Object.defineProperty(obj, "a", {
	value: 2,
	writable: false,
	configurable: true,
	enumerable: true
})

obj.a = 3

obj.a // 2

可以看到，writable设置为false后即使我们对属性进行了修改操作，也是静默失败（silently failed）了。而如果在严格模式下，则会抛出TypeError错误。

TypeError 错误表示我们无法修改一个不可写的属性。

configurable

configurable决定属性是否可配置，也就是是否可以用defineProperty(..)来改写特性。

var obj = {
  a: 2
}

obj.a = 3
obj.a // 3

Object.defineProperty(obj, "a", {
	value: 4,
	writable: true,
	configurable: false,
	enumerable: true
})

obj.a // 4
obj.a = 5
obj.a // 5

Object.defineProperty(obj, "a", {
	value: 6,
	writable: true,
	configurable: true,
	enumerable: true
}) // TypeError

最后一个defineProperty(..)会产生一个TypeError错误，不管是不是处于严格模式下。可以看到我们无法修改属性的特性，但是仍然可以修改属性的值。修改configurable的时候要要注意，一旦修改就无法撤回。

除了无法修改，删除操作也会失败。

var obj = {
  a: 2
}

obj.a // 2
delete obj.a;
obj.a; // undefined

Object.defineProperty(obj, "a", {
	value: 3,
	writable: true,
	configurable: false,
	enumerable: true
})

obj.a // 3
delete obj.a;
myObject.a; // 3

可以看到最后一个delete语句静默失败了。

如果属性保存的是引用，如果某个属性是某个对象/函数的最后一个引用者，那么delete删除后该引用就没有用了，可以被垃圾回收。但是，不要把delete看作是一个释放内存的工具。

enumerable

enumerable决定属性是否可枚举。

比如说，for..in 循环。如果把enumerable设为false，那么这个属性就不会出现在枚举中，但是仍然可以正常访问它。

var obj = {
  a: 2,
  b: 3,
  c: 4
}

obj.a // 2

Object.defineProperty(obj, "a", {
  value: 5,
	writable: true,
	configurable: true,
	enumerable: false
})

Object.keys() // ['b', 'c']
obj.a // 5

不变性

有的时候，我们希望自己创建的对象或者对象属性不被他人修改。我们就可以通过上面学的特性来进行设置。但是需要注意一点就是，所有方法创建的都是浅不变性，也就是说，它们只会影响目标对象直接相关的属性，如果是一些引用，我们还是可以修改其内部的值。

// 假设对象所有属性都设置为不可变
myObject.foo // [1,2,3]
myObject.foo.push(4)
myObject.foo // [1,2,3,4]

但是为了保护它的内容 myObject.foo，我们还需要使用下面的方法进一步让 foo 也不可变。

对象常量

结合 writable: false 和 configurable: false 就可以创建一个真正的常量属性（不可修改、重定义或者删除）

var myObject = {}
Object.defineProperty(myObject, "FAVORITE_NUMBER", {
	value: 42,
	writable: false,
	configurable: false
})

禁止拓展

如果我们想禁止一个对象添加新属性并且保留原有属性，可以通过使用**Object.preventExtensions(..)**来达到目的

var myObject = {
	a:2
}

Object.preventExtensions(myObject);

myObject.b = 3
myObject.b // undefined

在非严格模式下，创建属性 b 会静默失败。在严格模式下，将会抛出 TypeError 错误。

密封

如果我们需要对象不能添加新属性，也不能重新配置或者删除任何现有属性。我们可以使用Object.seal(..)**。它会创建一个“密封”对象，这个方法实际上就是在一个现有对象上调用Object.preventExtensions(..)** 并把所有现有属性标记为 configurable:false。不过“密封”后我们依然可以修改属性的值

冻结

如果我们需要更高的不可变性，我们就需要 Object.freeze(..) 。这个方法实际上就是在一个现有对象上调用 Object.seal(..) 并且把所有属性都设置为 writable:false，这样我们就无法修改它。

这个方法是我们可以应用在对象上的级别最高的不可变性。如果我们要求对属性中引用也是不可变，最好的方法就是遍历出那些引用，然后在这些对象上调用 Object.freeze(..) 。

顺道提一下，在ES6中，新增了代理（Proxy）。用于创建一个对象的代理，从而实现基本操作的拦截和自定义。我们也可以运用代理的方法来实现我们需要的不可变性，这里就不详细展开说了。

访问描述符

对象属性有两种表现形式，一种就是属性描述符，另一种就是访问描述符。两者是互斥的，只能选一。

访问描述符就是属性定义 getter 和 setter ，JavaScript会忽略它们的value和writable 特性，取而代之的是关心 set 和 get（还有 configurable 和 enumerable）特性。

Getter和Setter

看代码

var myObject = {
  // 给a定义一个getter
  get a() {
    return 2
  }
}

Object.defineProperty(myObject,"b", {
  // 给 b 设置一个 getter
  get: function() { return this.a * 2},
  // 确保 b 会出现在对象的属性列表中
  enumerable: true
})

myObject.a // 2
myObject.b // 4

myObject.a = 4
myObject.a // 2

这里可以看到，我们通过 getter 来设置一个a属性，也通过defineProperty来创建一个新属性，并且都可以获取到返回值。但是，我们无法对a进行赋值操作。那是因为 a 属性的返回值已经写死了，所以 set 操作没有意义。

为了让属性更合理，我们通常都是 getter 和 setter 同时设置的。

var myObject = {
	// 给 a 定义一个 getter
	get a() {
		return this.newa;
	},
	// 给 a 定义一个 setter
	set a(val) {
		this.newa = val * 2;
	}
}
myObject.a = 2;
myObject.a; // 4

这里的newa只是一个存储 a 值的另一个变量，它的名称没有讲究。

存在性

如果我们访问一个不存在的属性，返回值是undefined。那么有个问题，如果这个值可能是属性存储的undefined，那如何分清是不存在还是存储undefined？

我们可以通过两种方式来判断对象属性是否存在

var obj = {
	a: 2
}

("a" in obj) // true
("b" in obj) // false

obj.hasOwnProperty("a") // true
obj.hasOwnProperty("b") // false

in 操作符会检查属性是否在对象及其[[Prototype]]原型链中。而hasOwnProperty(..)只检查属性是否在对象中，不会检查[[Prototype]]原型链。

如果对象没有连接到Object.prototype，我们就无法使用hasOwnProperty(..)。这时候，我们可以通过 Object.prototype.hasOwnProperty. call(obj, “a”) 显式绑定到对象上。

对象的遍历

我们通过for..in 循环来遍历对象的可枚举属性列表，得到的是对象的键名。

var obj = {
	a: 3,
	b: 4,
	c: 5
}

for(var item in obj) {
	console.log(item)
}
// a b c

iterator

只要是遍历，都离不开迭代器。数组有内置的@@iterator，我们手动调用它，看看它是如何工作的。

var myArray = [1, 2, 3]
var it = myArray[Symbol.iterator]()

it.next() // { value: 1, done: false }
it.next() // { value: 2, done: false }
it.next() // { value: 3, done: false }
it.next() // { done: true }

我们使用ES6中的符号 Symbol.iterator 来获取对象的 @@iterator 内部属 性。引用类似 iterator 的特殊属性是要使用符号名，而不是符号包含的值。此外，虽然看起来很像一个对象，但 @@iterator 本身并不是一个迭代 器对象，而是一个返回迭代器对象的函数——这点非常精妙并且非常重要。

如我们所见，调用迭代器的next()返回一个包含value和done属性的对象。只有在遍历完最后一个数组元素再执行一次next()，done才为true。

普通对象不像数组，它没有内置的 @@iterator ，所以用不了 for..of 来遍历。之所 以要这样做，有许多非常复杂的原因，不过简单来说，这样做是为了避免影响未来的对象类型。

当然，我们可以给任何想要遍历的对象定义 @@iterator 

var myObject = {
  a: 2,
  b: 3
}

Object.defineProperty(myObject, Symbol.iterator, {
  enumerable: false,
	writable: false,
	configurable: true,
	value: function() {
		var o = this;
		var idx = 0;
		var ks = Object.keys(o);
		return {
			next: function() {
				return {
						value: o[ks[idx++]],
						done: (idx > ks.length)
				}
			}
	}
})
  
// 手动遍历 myObject
var it = myObject[Symbol.iterator]();
it.next(); // { value:2, done:false }
it.next(); // { value:3, done:false }
it.next(); // { value:undefined, done:true }

// 用 for..of 遍历 myObject
for (var v of myObject) {
	console.log( v );
}
// 2
// 3

for..of循环每次调用迭代器对象的方法时，内部的指针都会向前移动并返回属性列表的下一个值。

总结

这篇文章我们主要了解一些对象从未了解过的一些细节，例如属性中属性描述符和访问描述符的区别与应用，这个在今后的开发中一定用得上。而且流行的框架Vue也是通过defineProperty实现数据响应式的。相信如果以后有机会参与制作更底层的程序开发，这些知识一定用得上。
